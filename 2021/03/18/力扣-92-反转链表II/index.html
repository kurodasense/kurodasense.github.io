力扣：92.反转链表II


先让我秀一波提交记录图。因为这是我第一次靠自己想出来并且通过的第一道难度中等题。芜湖！

题目描述：

给你单链表的头指针 head 和两个整数left 和 right ，其中left <= right 。
请你反转从位置 left 到位置 right 的链表节点，返回 反转后的链表 。

示例1：

示例2：
提示：
  • 链表中节点数目为n
  • 1 <= n <= 500
  • -500 <= Node.val <= 500
  • 1 <= left <= right <= n
进阶：你可以使用一趟扫描完成反转吗？

我的题解：

看示例1可知，输入是一个头指针head、反转的起始位置left和结束位置right。要反转数字的话，
那就肯定要记录反转前的数字嘛。所以我就想到了用栈去做。
我们定义一个栈Stack<Integer> st = new Stack<Integer>()，并且在第一次循环中，将从left到
right的数字都押入到栈中。那示例1作为例子，第一次循环结束后，栈内应该是这样的：
stack = [2, 3, 4]。
显然，当栈stack执行pop()操作的话，是把顶部的元素弹出的。所以，我们把stack全部pop()完的话，
元素的弹出顺序就应该是[4, 3, 2]。
看！这是不是就是反转后的链表！
所以，我们只要再进行第二次循环，把从left到right位置的值用stack.pop()重新赋值就可以完成
链表的反转了。
还有一点，因为left和right给的值的意思是链表的位置。比如，left = 2和right = 4，是指从链表的第2个位置
到第4个位置之间的3个结点反转。所以我就设了一个count来计数得出目前所在的位置。（一开始我没注意到这个问题，
弄得我两次都提交出错了，，，）
